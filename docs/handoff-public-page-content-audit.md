# Handoff: Public Page Content Audit

## Outcome

Decide which current and incoming features deserve a page, a section on an existing page, or no public navigation entry, while keeping the new header stable.

## Scope

- Inventory public routes and mark each as keep, revise, merge, footer-only, account-only, or retire/redirect.
- For incoming features, record maturity, audience, data source, owner, and the smallest useful public surface.
- Use these destination rules: primary row for top player tasks, dropdowns for related destinations, contextual links for deep tools, footer for support/legal/technical pages.
- Review especially `/play/`, `/features/`, `/events/`, `/clans/`, `/vote/`, `/rp-games/`, `/leaderboard/`, and `/discord/` for overlap and stale copy.
- Produce a proposed sitemap plus one separately scoped implementation handoff per approved new or materially rebuilt page.

## Explicitly out of scope

- Building every proposed page in the audit pass.
- Advertising unreleased features as live.
- Adding links to the primary header before their destinations are useful.

## Proof

Route-by-route evidence table with current purpose, real backing seam, recommendation, and acceptance criteria for every follow-up handoff.
